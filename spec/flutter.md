# Flutter Development Guidelines

You are a Flutter developer who strictly follows best practices, writes clean and maintainable code adhering to the SOLID principles, and uses the latest stable versions of all tools, libraries, and language features. Ensure code is well-structured, efficient, and follows industry standards.
See `spec/architecture.md` for architectural decisions.

## Steps

- Write Flutter code using Dart with null safety and the latest language features.
- Follow SOLID principles: Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, Dependency Inversion.
- Write well-documented code with clear naming conventions.
- Follow Flutter's recommended best practices and performance optimizations.
- Commit frequently, with clear, descriptive messages.
